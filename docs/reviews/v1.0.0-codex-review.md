## P0 — Blockers

### #1 marketplace.json invalid — FIXED in Phase A
- **Codex finding:** `.claude-plugin/marketplace.json` was plugin-scoped metadata, not a marketplace catalog. `claude plugin validate .` failed (missing `owner`, missing `plugins[]`, unrecognized keys).
- **Resolution:** Rewrote to true marketplace catalog shape with `owner.email = sjh7877@naver.com` and `plugins[0].source = {source: "github", repo: "jaehoonE7877/rubrix"}`.
- **Evidence:** `claude plugin validate .` → `✔ Validation passed`.

### #2 hooks/hooks.json wrong shape — FIXED in Phase A
- **Codex finding:** Flat event→script map; Claude Code expects 3-level nesting with top-level `hooks` object containing event arrays of matcher/handler groups.
- **Resolution:** Rewrote to spec shape: `{ hooks: { EventName: [{ matcher: "", hooks: [{ type: "command", command: "${CLAUDE_PLUGIN_ROOT}/scripts/<name>.sh" }] }] } }` for all 7 events.
- **Evidence:** `node -e 'JSON.parse(...)'` parses; manifest shape matches docs at https://code.claude.com/docs/en/hooks.

### #3 hook block contract mixed JSON+exit2 — FIXED in Phase B
- **Codex finding:** `cli/src/commands/hook.ts` emitted JSON to stdout AND exited 2 for any block; PreToolUse should emit `hookSpecificOutput.permissionDecision` (allow/deny) at exit 0, and Stop should use the exit-code path.
- **Resolution:** Split emit into per-event helpers (`emitPreToolUse` writes `hookSpecificOutput.permissionDecision`, `emitContextOnly` writes `systemMessage`/`additionalContext`, `emitStop` uses stderr+exit2 on block).
- **Evidence:** `cli/tests/hook-emit.test.ts` (9 new tests) — all pass.

## P1 — Must-fix

### #4 locks.plan not checked for code edits — FIXED in Phase D
- **Codex finding:** `cli/src/hooks/handlers.ts` blocked code edits only until `locks.matrix=true`; missed `locks.plan` gate. Test at `cli/tests/hook.test.ts:81` encoded the wrong behavior.
- **Resolution:** Added `if (!locks.plan) return block` for `Edit`/`Write`/`MultiEdit`/`NotebookEdit`. Updated the test to expect `block` at MatrixLocked + plan=false, plus added a positive test for `PlanLocked + Edit → allow`.
- **Evidence:** `cli/tests/hook.test.ts` 19 tests pass; `skills/plan/SKILL.md` postcondition now matches handler.

### #5 /score gating wired to wrong event — FIXED in Phase D
- **Codex finding:** `tool === "SlashCommand"` check inside PreToolUse; direct `/score` invocation is `UserPromptExpansion`. `handleUserPromptExpansion` never blocked.
- **Resolution:** Added `promptInvokesScore()` helper used by both PreToolUse and UserPromptExpansion. UserPromptExpansion now emits `decision: "block"` with reason when score is invoked at `locks.plan=false`. Removed the redundant `tool === "SlashCommand"` clause.
- **Evidence:** New tests in `cli/tests/hook-emit.test.ts` cover both event paths.

### #6 Skill names not namespaced — FIXED in Phase H
- **Codex finding:** `PLUGIN-README.md` and `README.md` documented `/rubric`, `/matrix`, `/plan`, `/score`; Claude Code plugin namespace exposes them as `/rubrix:rubric` etc.
- **Resolution:** Updated `README.md`, `PLUGIN-README.md`, `CLAUDE.md`, all 4 SKILL.md headers + cross-references to `/rubrix:<name>`. Added namespaced trigger entries to `SCORE_TRIGGERS` set in `cli/src/hooks/handlers.ts` (Phase D).

### #7 SKILL.md `verify` ↔ schema `method` drift — FIXED in Phase C
- **Codex finding:** All 4 SKILL.md taught `verify` field; `cli/schemas/rubrix.schema.json` matrix.rows allowed only `method` (with `additionalProperties: false`); rubric.criteria didn't allow `verify` at all.
- **Resolution:** Renamed schema `matrix.rows[].method` → `verify`; added `verify` to `rubric.criteria[]`. Updated `examples/ios-refactor/rubrix.json` (the one example using `method`) to `verify`.
- **Evidence:** Both examples validate; new vitest cases reject `method` and accept `verify`.

### #8 Lock command lacks semantic validation — FIXED in Phase E
- **Codex finding:** `cli/src/commands/lock.ts` only checked artifact presence; could lock a matrix referencing nonexistent `criterion` ids, or a plan with bogus `covers` ids.
- **Resolution:** Created `cli/src/core/integrity.ts` with `checkMatrixIntegrity()` and `checkPlanIntegrity()` — verifies criterion refs, matrix-row refs, no duplicate ids, full coverage. Lock command runs the integrity check before transitioning state.
- **Evidence:** `cli/tests/integrity.test.ts` (8 new tests) + 5 new lock semantic tests in `cli/tests/lock.test.ts`.

### #9 Docs/version still v0.1 scaffold — FIXED in Phase H
- **Codex finding:** `PLUGIN-README.md` line 1 said "v0.1"; `README.md` / `CLAUDE.md` said "초기 스캐폴드"; `cli/package.json` / `.claude-plugin/plugin.json` were `0.1.0`.
- **Resolution:** Bumped all manifests to `1.0.0` (cli, plugin, marketplace, registry catalogs); rewrote `README.md`, `PLUGIN-README.md`, `CLAUDE.md`, `VERIFICATION.md` to describe the production v1.0 surface; added `> Status (v1.0.0)` header to `docs/extensible-plan.md`. GitHub URLs updated to `jaehoonE7877/rubrix`.

## P2 — Polish

### #10 saveContract not atomic / symlink-safe — FIXED in Phase F
- **Codex finding:** `cli/src/core/contract.ts` used `writeFileSync` directly; crash mid-write could corrupt state; symlinks were followed (overwriting unintended targets).
- **Resolution:** Rewrote `saveContract` as same-directory temp file → `fsync` → `renameSync(tmp, realpath(path))`. Resolves symlinks to the real file before writing so the symlink itself is preserved.
- **Evidence:** `cli/tests/contract.test.ts` (3 tests) — no leftover temp files, symlinks preserved, schema-invalid contract rejected.
